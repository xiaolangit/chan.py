# 三种线段算法流程图

## 1. Chan算法流程图

```
开始
  ↓
删除末尾不确定线段 (do_init)
  ↓
创建上升/下降特征处理器
  ↓
遍历笔数据
  ↓
┌─────────────────────────────────┐
│ 对每个笔进行处理                    │
│ ┌─────────────────────────────┐ │
│ │ 选择特征处理器                │ │
│ │ - 下降笔 → 上升线段处理器      │ │
│ │ - 上升笔 → 下降线段处理器      │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 添加笔到特征序列             │ │
│ │ - 第一元素：直接添加          │ │
│ │ - 第二元素：尝试合并          │ │
│ │ - 第三元素：检查分形          │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 分形形成？                   │ │
│ │ - 是：处理分形特征            │ │
│ │ - 否：继续下一个笔            │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
  ↓
处理分形特征 (treat_fx_eigen)
  ↓
┌─────────────────────────────────┐
│ 判断线段是否可以结束             │
│ - 有缺口：寻找反向分形确认        │
│ - 无缺口：直接确认               │
└─────────────────────────────────┘
  ↓
添加确定线段
  ↓
收集剩余线段
  ↓
结束
```

## 2. DYH算法流程图

```
开始
  ↓
初始化处理 (do_init)
  ↓
计算确定线段 (cal_bi_sure)
  ↓
┌─────────────────────────────────┐
│ 遍历笔数据                       │
│ ┌─────────────────────────────┐ │
│ │ 检查基本条件                 │ │
│ │ - 至少3个笔                 │ │
│ │ - 方向连续性                │ │
│ │ - 突破起始笔范围             │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 应用双情况判断               │ │
│ │ - situation1: 弱突破        │ │
│ │ - situation2: 强突破        │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 满足条件？                   │ │
│ │ - 是：添加新线段             │ │
│ │ - 否：继续下一个笔           │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
  ↓
尝试更新最后线段 (try_update_last_seg)
  ↓
检查剩余笔突破 (left_bi_break)
  ↓
计算不确定线段 (cal_bi_unsure)
  ↓
收集剩余线段
  ↓
结束
```

## 3. Def算法流程图

```
开始
  ↓
初始化处理 (do_init)
  ↓
计算确定线段 (cal_bi_sure)
  ↓
┌─────────────────────────────────┐
│ 遍历笔数据                       │
│ ┌─────────────────────────────┐ │
│ │ 峰值笔管理                   │ │
│ │ - 更新同方向峰值笔           │ │
│ │ - 维护当前最极端笔           │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 简单高低点比较               │ │
│ │ - is_up_seg: 高点突破       │ │
│ │ - is_down_seg: 低点突破     │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 线段条件判断                 │ │
│ │ - 峰值笔存在且方向改变       │ │
│ │ - 满足最小间隔要求           │ │
│ └─────────────────────────────┘ │
│               ↓                 │
│ ┌─────────────────────────────┐ │
│ │ 添加线段                     │ │
│ │ - 以峰值笔为终点             │ │
│ │ - 更新峰值笔                 │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
  ↓
处理最后峰值笔
  ↓
收集剩余线段
  ↓
结束
```

## 4. 关键决策点对比

### Chan算法决策点
1. **分形识别**：三个特征元素是否构成有效分形
2. **合并处理**：元素间是否可以合并
3. **突破确认**：是否存在实际突破
4. **结束判断**：缺口处理和反向分形搜索

### DYH算法决策点
1. **情况判断**：situation1或situation2是否满足
2. **方向连续**：笔的方向是否与线段连续
3. **突破检查**：是否突破起始笔范围
4. **极值更新**：寻找最极端的笔作为终点

### Def算法决策点
1. **高低点比较**：简单的高低点突破判断
2. **峰值管理**：维护和更新峰值笔
3. **方向变化**：检测峰值笔方向变化
4. **间隔要求**：满足最小笔间隔

## 5. 算法特征总结

| 特征 | Chan算法 | DYH算法 | Def算法 |
|------|----------|---------|---------|
| **理论基础** | 缠论特征序列 | 笔突破关系 | 高低点比较 |
| **核心机制** | 分形特征识别 | 双情况判断 | 峰值笔管理 |
| **计算复杂度** | 高（O(n²)） | 中（O(n)） | 低（O(n)） |
| **准确性** | 最高 | 中等 | 较低 |
| **实现复杂度** | 复杂 | 中等 | 简单 |
| **适用场景** | 理论研究、精确分析 | 实时交易、平衡需求 | 快速判断、大批量处理 |

每种算法都有其独特的设计理念和适用场景，可以根据具体需求选择最合适的算法。
